# 60. K Closest Points to Origin

https://leetcode.com/problems/k-closest-points-to-origin/description/

You are given an 2-D array `points` where `points[i] = [xi, yi]` represents the coordinates of a point on an X-Y axis plane. You are also given an integer `k`.

Return the `k` closest points to the origin `(0, 0)`.

The distance between two points is defined as the Euclidean distance `(sqrt((x1 - x2)^2 + (y1 - y2)^2))`.

You may return the answer in any order.

# Solution

Key insight: Using `sqrt(x^2 + y^2)` is unnecessary, and introduces floating-point precision issues. Since we are comparing relative distances, the squared distance preserves ordering, and avoids computing the square root.

```py
class Solution:
    def kClosest(self, points: List[List[int]], k: int) -> List[List[int]]:
        distances = []

        for x, y in points:
            current_square_distance = x**2 + y**2
            heapq.heappush(distances, (-current_square_distance, (x, y)))
            if len(distances) > k:
                heapq.heappop(distances)

        return [point for _, point in distances]
```

For $N$ points and $K$:
- Time complexity: $O(N \log K)$
- Space complexity: $O(K)$
