# 41. Merge K Sorted Lists

https://leetcode.com/problems/merge-k-sorted-lists/

You are given an array of `k` linked lists `lists`, where each list is sorted in ascending order.

Return the sorted linked list that is the result of merging all of the individual linked lists.

# Solution

Key insight: For using a min-heap, we need to implement the `__lt__` method, so that the smallest node can be placed on the top. We create a `NodeWrapper` class to help do that. We take each list, and push it onto the heap. While we have lists, we take the list with the smallest value in front, append that to our final list, and then push the next part of the list back into the min-heap, maintaining the heap's invariant.

```py
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next

class NodeWrapper: # Used to implement __lt__ for the nodes, to allow adding to the min-heap
    def __init__(self, node: Optional[ListNode]):
        self.node = node

    def __lt__(self, other):
        return self.node.val < other.node.val

class Solution:    
    def mergeKLists(self, lists: List[Optional[ListNode]]) -> Optional[ListNode]:
        # Create the heap
        heap = []
        for l in lists:
            heapq.heappush(heap, NodeWrapper(l))

        dummy = curr = ListNode() # Create the final list we want to return
        while len(heap) > 0: # While we still have elements to process
            l = heapq.heappop(heap) # Take the list with the smallest valued head
            curr.next = l.node # Add the node to the final list
            curr = curr.next # Move forward

            l = l.node.next 
            if l: # If there are still remaining elements in the current list
                heapq.heappush(heap, NodeWrapper(l)) # Append the list back into the heap

        return dummy.next # Return the created list
```

For $K$ `lists`, and $N$ nodes total across the $K$ lists:
- Time complexity: $O(N \log K)$ (We have to go through all $N$ nodes, and the heap has maximum size $K$, so each push/pop operation onto the heap is $O(\log K)$)
- Space complexity: $O(K)$ additional space (size of `heap`)
