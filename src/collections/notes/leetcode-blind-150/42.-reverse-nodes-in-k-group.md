# 42. Reverse Nodes in K-Group

https://leetcode.com/problems/reverse-nodes-in-k-group/description/

You are given the head of a singly linked list `head` and a positive integer `k`.

You must reverse the first `k` nodes in the linked list, and then reverse the next `k` nodes, and so on. If there are fewer than `k` nodes left, leave the nodes as they are.

Return the modified list after reversing the nodes in each group of `k`.

You are only allowed to modify the nodes' next pointers, not the values of the nodes.

# Solution

```py
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next

class Solution:
    def reverse(self, head):
        if not head:
            return head

        prev = None
        curr = head
        while curr:
            temp = curr.next
            curr.next = prev
            prev = curr
            curr = temp

        return prev

    def getKth(self, head: Optional[ListNode], k: int) -> Optional[ListNode]:
        curr = head
        while curr and k > 0:
            if not curr:
                return curr

            curr = curr.next
            k -= 1

        return curr


    def reverseKGroup(self, head: Optional[ListNode], k: int) -> Optional[ListNode]:
        dummy = ListNode(0, head)
        prev_group_end = dummy # Represents the node right before the current k-group

        while True:
            current_group_end = self.getKth(prev_group_end, k) # Get the k-th node after the previous group's end
            if not current_group_end: # If there is no node, then the next group is smaller than k, and we can stop
                break

            next_group_start = current_group_end.next # The start of the next k-group
            current_group_start = prev_group_end.next # The start of the current k-group

            current_group_end.next = None # Break the link between the current k-group and the next k-group, so we can reverse the current k-group
            prev_group_end.next = self.reverse(current_group_start) # Reverse the current k-group, and attach it to the previous k-group
            current_group_start.next = next_group_start # Attach the current k-group's original start (which is now the end of the current k-group) to the next k-group

            prev_group_end = current_group_start # Move the prev_group_end pointer to the current k-group's end

        return dummy.next
```

For a list of $N$ nodes:
- Time complexity: $O(N)$
- Space cmoplexity: $O(1)$
